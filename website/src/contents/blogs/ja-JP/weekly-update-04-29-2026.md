スプリングクリーンミュー。

Eiger 由来の複数監査項目に対応し、CLI をリファクタし、マイナーとインデクサー横断の不具合を修正し、新コンパクトアグリゲーター上でワームホール統合テストを走らせた。

ZK では、ワームホールフローが先週出荷した 2x8 コンパクトアグリゲーターをクリーンに通過するようになった。プローバ性能を倍にしたアーキテクチャであり、より速くスケーラブルなプライベート取引につながる。ユーザー向けにはモバイルウォレット 1.3.4 と 1.3.5、GPU 不具合修正入りの Quantus マイナー v3.1.0 を出荷。マイニングから報酬請求までをカバーするようドキュメントも更新した。

Network School でプランク・テストネット向け初のマイナーワークショップを開催した。

今週の成果物は以下のとおり。

### Core Tech & ZK

- Eiger からの複数監査項目に対応。
- 2x8 コンパクトアグリゲーターでワームホール統合テストを実行。
- 互換性・トランザクション・ライフサイクル・手数料見積もりの課題に対処するため Quantus CLI をリファクタ。

### ネットワーク & インフラ

- プランク・テストネットでストレステストを実施。
- Subsquid インデクサーの性能改善と CPU スパイク修正。今週は Subsquid がモバイルアプリ UX（残高が読み込めない）の原因となったが解消済み。
- GPU マイニング不具合修正の Quantus マイナー v3.1.0 をリリース。

### Web & モバイルアプリ

- モバイルウォレット 1.3.4 と 1.3.5 を出荷。
- モバイルウォレットのデザイン改善。次はオンボーディングフロー。
- アカウント管理、設定、オンボーディング、ホーム画面スケルトン、POS 有効化、空状態、マイナー報酬の各フローをウォレット向けに設計。
- ウォレット向けワームホールフロー UX をマッピング。
- [docs.quantus.com](https://docs.quantus.com) を更新し、マイニングから報酬請求までを E2E でカバー。
- CLI、モニタリング、チェーン、ウェブサイト、ZK 回路各リポの deepwiki を更新。
