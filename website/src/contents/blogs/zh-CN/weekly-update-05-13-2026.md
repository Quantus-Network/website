早期的比特币矿工可以用笔记本电脑每天挖出数千个 BTC。

比特币 CPU 挖矿基本上只维持了 2 年的生命力。

随着网络的发展，算力也随之增长。GPU 挖矿变得更可行，然后是 ASIC。

当你第一次听说 BTC 时，在消费级硬件上挖矿已经不可能了。

当时没有可以购买的交易所。

由于没有二级市场，也就没有“价格”。

为了得到一个“价格”，你需要估算每枚代币的电力成本。

当你可以在市场上购买时，大部分代币已经分发完毕。

比特币在头四年里发行了超过一半的代币。

全部都流向了早期矿工。

这就是“公平启动”（Fair Launch）的标准。

我们或许可以做得更好。

在我们准备启动 [@QuantusNetwork](https://x.com/@QuantusNetwork) 时，我们优先考虑让挖矿从第一天起就触手可及。

你今天只需点击几下，就可以在我们的测试网上运行挖矿节点。

上周我们更新了 GUI 矿工应用。即使你不是技术人员，也可以为第一天的主网挖矿做好准备。

### Core Tech & ZK (核心技术与零知识证明):

- 创建了矿工端委托 ZK 证明的原型。这一实验性功能将允许矿工选择作为额外角色参与 Wormhole 交易的证明聚合。
- 解决了 HashCloak 和 Eiger 的审计发现。

### Network & Infra (网络与基础设施):

- 发布了带有 Planck 测试网赎回和余额功能的 GUI 矿工更新。下载和操作指南请访问 docs (dot) quantus (dot) com
- 在所有 pull request 中部署了基于 AI 的安全审计框架。完成了首次自动化的全链安全审查。
- 对 RPC 节点进行了渗透测试。

### Web & Mobile App Updates (网页与移动端更新):

- 在钱包中实现了矿工奖励界面和销售点 (POS) 界面。
- 优化了昂贵的浏览器查询并更新了统一的账户事件。
- 移动端应用修复。
