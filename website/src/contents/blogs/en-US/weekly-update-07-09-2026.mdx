Every chain says they'll become post-quantum.

So we built a scoreboard.

[Migration.fail](https://migration.fail/) tracks the percentage of crypto market cap that runs on quantum-resistant cryptography.

Right now it's 0.046%.

Basically zero.

Over $2T has to move from pre-quantum addresses and become post-quantum.

So we're building the destination.

We shipped v1.5.5 of the Quantus app with Keystone hardware wallet support.

Now QUAN has both multisig and cold storage support.

It's all open source on GitHub.

The whole industry is going to need to go post-quantum, now you can watch the real time progress at [migration.fail](https://migration.fail/).

Here's what else we built:

### Github Activity (22 merged pull requests):

![Github Stats](/blog/assets/github-stats-weekly-update-07-09-2026.webp)

### Core Tech & ZK

- Merged second layer proof aggregation in the ZK circuits, letting us aggregate already-aggregated proofs for another step up in throughput.
- Cut peak memory for ML-DSA-87 signing by streaming the key matrix, which makes Dilithium practical on embedded hardware like the Keystone.
- Hardened threshold signature resharing and improved constant-time behavior in our Dilithium library.
- Cleared medium and low findings from our internal chain review, hardening FRAME primitives and dependencies.
- Fixed additional internal review findings in the Poseidon hashing code.
- Fixed orphaned held assets on reversible-transfer recovery in the chain.
- Added a quantus exercise smoke and fuzz test suite to the Quantus CLI, plus an update feature.

### Network & Infra

- Released miner v3.3.0 and v3.3.1 with improved Windows GPU support and GPU adapter selection fixes for multi-backend duplication and out-of-memory errors.
- Improved the mining setup flow and added a mining setup script to the docs site.
- Added tech governance event support to Subsquid

### Web & Mobile App Updates

- Shipped v1.5.5 of the mobile app to the App Store with Keystone hardware wallet support: firmware updates, multisig support in the firmware, security hardening of key handling, and an integration bug fix. Our firmware fork is open source on GitHub.
- Cleared all medium findings from our internal Keystone firmware review, adding chain conformance tests, hedged signing, and device binding.
- Launched [migration.fail](https://migration.fail/).
- Added multiple seed phrase support to the mobile wallet.
- Refactored the send flow and shipped UX fixes across the mobile app.
- Hardened the copy seed phrase flow and transaction parsing, and added haptic feedback to amount input.
- Shipped correctness fixes from the internal v12 runtime review.
