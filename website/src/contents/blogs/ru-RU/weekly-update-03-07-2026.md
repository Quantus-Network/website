Вы сжигаете монеты на нетратимый адрес.

ZK-доказательство минтит их в новом месте.

След исчезает.

[@QuantusNetwork](https://x.com/@QuantusNetwork) называет это «вормхолл»-транзакциями (транзакциями через червоточину).

Вы отправляете монеты на вормхолл-адрес, который выглядит идентично любому другому счету в реестре. Но этот адрес математически невозможно потратить. Чтобы вывести средства, вы доказываете знание секрета с помощью ZK-доказательства, и чейн минтит эквивалентные монеты на новый адрес выхода.

Никто не может связать вход с выходом.

И вот здесь начинается самое интересное для майнеров.

Протокол требует, чтобы майнеры предоставляли 32-байтовый прообраз (preimage) для своего адреса вознаграждения. Чейн хеширует его для создания адреса.

Публичные ключи Dilithium массивны. Они не помещаются в 32 байта. Таким образом, прообраз не может быть реальным публичным ключом, и полученный адрес гарантированно является нетратимым вормхолл-адресом.

Конфиденциальность обеспечивается протоколом, без необходимости какого-либо согласия (opt-in).

На этой неделе мы встроили это в приложение для майнеров. В следующем тестнете все вознаграждения за блок по умолчанию будут направляться через вормхолл-адреса.
